Project Library of Babel (v0.1)
# 1. 概要
Project Library of Babel は、情報を保存するのではなく、決定論的なハッシュ関数によって生成される無限の疑似乱数空間（辞書空間）の中の座標として再定義する、実験的な暗号化・圧縮システムです。 物理的なデータの断片を、ハッシュ関数の計算過程における出現位置に置換することで、情報の局所性を排除し、空間上の点へと変換します。

## 2. 構成要素
本システムは以下の4つの動的要素によって定義されます。

要素	定義	説明
S (Salt)	公開値	システムが生成する一意の16バイト乱数。辞書の個体差を生む。
P (Password)	秘密鍵	ユーザーが管理する鍵。辞書空間へのアクセス権限となる。
H(x)	ハッシュ関数	BLAKE3 を使用。高速かつ決定論的な256bit出力を利用。
D (Dictionary)	辞書空間	$H(x)$ を再帰的に適用することで生成される無限のバイト列。
## 3. 暗号化アルゴリズム (The Process)
Step 1: 初期シード生成
最初のハッシュ生成の起点となるシード値 $G_0$ を定義します。 $$G_0 = H(P + S)$$

Step 2: スキャンとマッチング
元データをチャンク $c_1, c_2, ..., c_n$ （デフォルト1バイト）に分割し、以下のループを実行します。

照合 (Matching): 現在のハッシュ値 $H_i$（32バイト）の内部に、チャンク $c_n$ と一致するバイトパターンがあるか走査します。
失敗時 (No Match):
ミスカウント（0.0）をインクリメントします。
ハッシュを更新します: $H_{i+1} = H(H_i)$
一致するまで繰り返します。
成功時 (Match):
一致したハッシュ内の開始位置インデックス $I$ を座標として記録します。
連続したミスカウントをRLE圧縮（例: x10_0.0）して出力バッファに書き込みます。
インデックス $I$ を書き込みます。
ベクトル化 (Vectorization):
次のハッシュ生成に座標情報を組み込み、辞書の経路を分岐させます。
$H_{next} = H(H_i + I)$
Step 3: 最適化
座標リストにおける連続する失敗（ハッシュの空振り）をランレングス形式で記録することで、マッチまでの距離が長い場合のデータ肥大化を抑制します。

## 4. アーカイブ構造 (Output Format)
最終的な出力は以下のセクションを持つテキスト形式（またはバイナリ）です。

text
[Header]
Version: ABGRUND INDEX 0.1
Hash: BLAKE3-256
Salt: <16進数 32文字>
ChunkSize: <数値>
[Body]
<座標リスト: 14, x8_0.0, 7, 22, ...>
[Footer]
Checksum: <元データのBLAKE3ハッシュ>
## 5. 復号ロジック (Decryption)
復号者はパスワード $P$ とヘッダーの $S$ を用いて、暗号化時と同一の辞書空間 $D$ をオンメモリで再現します。

座標解析: 座標リストを先頭から読み込みます。
スキップ: xN_0.0 の数だけハッシュ計算 $H_{i+1} = H(H_i)$ を実行し、辞書を進めます。
抽出: 座標 $I$ が指定されたハッシュに到達したら、その位置 $I$ から ChunkSize 分のデータを抽出します。
分岐: 抽出後、暗号化時と同様に $H_{next} = H(H_i + I)$ を行い、次の座標へ進みます。
検証: 結合されたデータの最終ハッシュを計算し、Footer の Checksum と照合します。
## 6. 特筆すべき特性
### 6.1 衝突（コリジョン）の許容
辞書内に同じデータパターンが複数存在する場合でも、本仕様では「最初に見つかった座標」を採用することで一意性を担保します。

### 6.2 スライディング・スケール
マッチングさせるチャンクサイズ $c$ を小さく（例: 1byte）すれば、辞書内で早期に見つかる確率が上がり、確実に「圧縮」が可能になります。大きくすれば、巨大なハッシュ計算の果てに「劇的な圧縮」が起きる可能性を秘めます。これは計算資源とデータサイズのトレードオフです。

### 6.3 情報の空間配置
本システムの最大の特徴は、情報の局所性（Locality）を捨て、空間上の位置に置換している点にあります。0.0 のカウントは「データが見つかるまでの期待値」を表現しており、数学的に美しい構造を持っています。

+# Library of Babel Crypt (v0.1)
+
+情報の局所性を捨て、無限のハッシュ空間上の「座標」へと置換する実験的暗号化ツール。
+
+---
+
+## Quick Start Guide
+
+### 1. インストール
+Rustがインストールされている環境でビルドします。
+```bash
+cargo build --release
+```
+
+### 2. 暗号化 (Encrypt)
+ファイル、パスワード、およびオプションのチャンクサイズを指定します。
+```bash
+./target/release/library_of_babel_crypt encrypt secret.txt mypassword123 1
+```
+*   `secret.txt.babel.txt` が生成されます。
+*   実行中、TUI（ターミナルUI）でハッシュスキャンの進捗がリアルタイム表示されます。
+
+### 3. 復号 (Decrypt)
+生成されたアーカイブファイルとパスワードを指定します。
+```bash
+./target/release/library_of_babel_crypt decrypt secret.txt.babel.txt mypassword123
+```
+*   `secret.txt.dec` が生成され、元のデータが復元されます。
+

+
+##  Chunk Size の仕様 (Sliding Scale)
+
+本システムでは、データを何バイトずつの塊（チャンク）として辞書から探すかを選択できます。
+
+| サイズ | 特徴 | 用途 |
+| :--- | :--- | :--- |
+| **1 byte** | マッチ確率が非常に高い（1/256）。スキャンが高速。 | 通常の暗号化・デモ |
+| **2 bytes** | マッチ確率が下がる（1/65536）。圧縮率が向上する可能性がある。 | バランス重視 |
+| **4 bytes+** | マッチは極めて稀。発見された場合、膨大なデータを数文字の座標に置換できる。 | 「奇跡」の圧縮を待つ場合 |
+
